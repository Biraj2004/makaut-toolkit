# Security Policy

## Supported Versions

This project is maintained on the default branch and latest release.

| Version | Supported |
| --- | --- |
| Latest release | Yes |
| master | Yes |
| Older versions | No |

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

1. Open a private report using GitHub Security Advisories (preferred).
2. If private reporting is unavailable, open a GitHub issue with minimal details and request a private follow-up channel.
3. Do not publicly share exploit details until a fix is available.

## What to Include

Please include:

- A clear description of the issue
- Steps to reproduce
- Expected vs actual behavior
- Affected files or routes
- Possible impact
- Optional mitigation suggestions

## Response Timeline

Target response windows:

- Initial acknowledgement: within 72 hours
- Triage update: within 7 days
- Fix or mitigation plan: as soon as possible based on severity

## Disclosure Policy

- We follow coordinated disclosure.
- A vulnerability is disclosed publicly only after a fix or mitigation is available.
- Credit will be given to reporters who want to be acknowledged.
